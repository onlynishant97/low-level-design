Repositories
«repository»
VehicleRepository
Stores vehicles; maintains a secondary location index for fast lookup by branch.
Methods
save(vehicle)
findAllActiveVehiclesByLocation(location)
⚡ Thread-safe
ConcurrentHashMap + CopyOnWriteArrayList Primary map is a ConcurrentHashMap<String, Vehicle>. The secondary location index uses CopyOnWriteArrayList per location key — prevents ArrayList.add() race conditions when multiple vehicles are onboarded concurrently.
«repository»
ReservationRepository
Persists reservations; supports lookup by ID and by vehicle for overlap checking.
Methods
save(reservation)
findById(id)
findByVehicle(vehicleId)
findAll() · delete(id)
⚡ Thread-safe
ConcurrentHashMap All reads and writes go through a ConcurrentHashMap. Overlap checks that read this map are always called from inside the per-vehicle ReentrantLock in ReservationService, so the compound read-then-write is also atomic at the service level.
«repository»
RentalRepository
Stores rental records keyed by rental ID; supports lookup by reservation.
Methods

⚡ Thread-safe
ConcurrentHashMap (key = rental.getId()) Fixed from the original bug where the key was reservationId. Now keyed on rental.getId(), ensuring correct isolation between rentals.

ReservationService
Core booking engine: validates availability, prevents double-booking, and manages reservation lifecycle.
Dependencies
ReservationRepository, PricingStrategy
Methods
initiateReservation(customer, vehicle, s, e, amount)
createReservation(reservationId, paymentId)
cancelReservation(reservation, paymentService, payment)
modifyReservation(reservation, newStart, newEnd)
⚡ Concurrency — primary guard
Per-vehicle ReentrantLock with 5s timeout A ConcurrentHashMap<vehicleId, ReentrantLock> provides one lock per vehicle. Both initiateReservation() and modifyReservation() call lock.tryLock(5, SECONDS) before reading the reservation table and mutating vehicle status. The critical section covers: (1) status check, (2) overlap query, (3) vehicle status update to RESERVED, (4) save. This makes the entire check-then-act atomic per vehicle without blocking unrelated vehicles. InterruptedException is handled by restoring the interrupt flag.

Key Relationships
From → To Type Notes
Vehicle → VehiclePricing Composition Vehicle owns its pricing config; no shared pricing objects
Vehicle → Location Association currentLocation tracks which branch the car is at
Reservation → Vehicle, Customer Association Holds live references; vehicle status changes are visible through the reference
Rental → Reservation Association One-to-one; rental is created only from an ACTIVE reservation
Bill → Rental Association Bill reads odometer and dates from Rental to compute charges
StandardPricingStrategy → PricingStrategy Implements Open for extension: add PromoPricingStrategy without touching services
CreditCardPaymentStrategy → PaymentStrategy Implements UPI, wallet, or net-banking strategies plug in the same way
ReservationService → ReservationRepository Uses (dependency) Overlap check reads from repo inside the ReentrantLock critical section
ReservationService → PricingStrategy Uses (dependency) Recomputes price server-side to prevent client-tampered amounts
PaymentService → PaymentStrategyFactory Uses (dependency) Factory resolves the correct strategy at runtime from the provider string
VehicleService → VehicleRepository, ReservationRepository Uses (dependency) searchVehicles combines both to return only date-available cars
DamageService → RentalRepository Uses (dependency) Persists damage flag on rental and mutates vehicle status to UNDER_MAINTENANCE
